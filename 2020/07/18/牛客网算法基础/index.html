牛客网算法基础

一、时间复杂度

常数时间的操作：

一个操作如果和数据量没有关系，每次都是固定时间内完成的操作，叫做常数操作。

时间复杂度为一个算法流程当中，常数操作数量的指标，常用O（读作big O）来表示。具体来说，在常数操作数量的表达式中，只要高阶项，不要低阶项，也不要高阶项的系数，剩下的部分如果记为f(N)，那么时间复杂度为O(f(N))。

评价一个算法流程的好坏，先看时间复杂度指标，然后再分析不同数据样本下的实际运行时间，也就是常数项时间。

一个简单的理解时间复杂度的例子：

一个有序数组A，另一个无序数组B，打印B中的所有不在A中的数，A数组长度为N，B数组长度为M。

算法流程1：对于数组B中的每一个数，都在A中通过遍历的方式找一下

时间复杂度 → O(M*N)

算法流程2：对于数组B中的每一个数，都在A中通过二分的方式找一下

时间复杂度 → $O(M*log_2N)$

算法流程3：先把数组B排序，然后用类似外排的方式打印所有不在A中出现的数

算法复杂度 → 排序：$O(M*log_2M)$ + 外排：O(N+M)

对数器的概念和使用

(1)有一个想要测的方法a

(2)实现一个绝对正确但是复杂度不好的方法b

(3)实现一个随机样本产生器

(4)实现比对的方法

(5)把方法a和方法b比对很多次来验证方法a是否正确

(6)如果有一个样本使得比对出错，打印样本分析是哪个方法出错

(7)当样本数量很多时比对测试依然正确，可以确定方法a已经正确

排序算法时间复杂度分析：

(1)冒泡排序细节讲解与复杂度分析：

每次比较替换相邻的两个数，小的数往上冒，每次循环都将最大的数排到最后一位

时间复杂度为 N + N-1 + N-2 + … + 1 → O(N2)，额外空间复杂度为O(1)

(2)选择排序的细节讲解与复杂度分析：

每次遍历所有数，把最小的数挑选出来放在数组的第一位

时间复杂度为 N + N-1 + N-2 + … + 1 → O(N2)，额外空间复杂度为O(1)

(3)插入排序的细节讲解与复杂度分析：

类似于扑克牌排序，数组的数都与前面的数比较，如果数组的数都比前面的数小就交换，交换后继续与前面的数比较，重复此操作直到前面的数小于数组插入的数，遍历完整个数组。

时间复杂度与数据状况相关，如果数组是正序排好序的，则时间复杂度为O(N)；如果数组为逆序排好序的，则时间复杂度为O(N2)，额外空间复杂度为O(1)。

(4)归并排序的细节讲解与复杂度分析

把数组分为左右两个序列先进行排序，然后再把左右两个序列利用双指针移动进行比较排序。

时间复杂度为O(N * logN)，额外空间复杂度为O(N)。

归并排序是创建在归并操作上的一种有效的排序算法，效率为O(nlogn)，该算法是采用分治法的一个经典类型应用，且各层分支递归可以同时进行。

分支法：

字面上的解释为“分而治之”，就是把一个复杂的问题分成两个或多个更多的相同的或相似的子问题，再把子问题分为更小的子问题…直到最后子问题可以简单的直接求解，原问题的解即子问题的解的合并。

分而治之的设计思想：

将一个难以解决的大问题，分割成一些规模较小的相同问题，以便各个击破，分而治之。

分而治之的基本步骤：

step1 → 分解：将原问题分解为若干个规模较小，相互独立，与原问题形式相同的子问题

step2 → 解决：若子问题规模较小而容易被解决则直接解，否则递归地解各个子问题

step3 → 合并：将各个子问题的解合并为原问题的解

依据分治法设计程序时的思维过程：

实际上就是类似于数学归纳法，找到解决问题的求解方程公式，然后根据方程公式设计递归程序。

1.一定是先找到最小问题规模时的求解方法

2.然后考虑随着问题规模增大时的求解方法

3.找到求解的递归函数式后（各种规模或因子），设计递归程序即可

def merge(a,b):
    c = []
    h = j = 0
    while j < len(a) and h < len(b):
        if a[j] < b[h]:
            c.append(a[j])
            j += 1
        else:
            c.append(b[h])
            h += 1
    if j == len(a):
        for i in b[h:]:
            c.append(i)
    else:
        for i in a[j:]:
            c.append(i)
    return c

归并排序之所以比前面几种排序算法快，是因为归并排序利用了每次排序的有效数组信息，而冒泡、选择、插入排序每次都是比较得出单个数的信息，剩余数组信息没有利用上， 需要重复比较排序，浪费循环次数。

归并排序应用：

(1)小和问题

在一个数组中， 每一个数左边比当前数小的数累加起来， 叫做这个数组的小和。 求一个数组
的小和。
例子：
[1,3,4,2,5]
1左边比1小的数， 没有；
3左边比3小的数， 1；
4左边比4小的数， 1、 3；
2左边比2小的数， 1；
5左边比5小的数， 1、 3、 4、 2；
所以小和为1+1+3+1+1+3+4+2=16

解法：

可以使用归并排序，在merge时，由于左右两部分都已经有序，可以确定一侧的数都大于正在比较的书，比如归并 1 3 7 | 2 4 5两个部分时，1比2小，此时可以确定2后面的数都比1大，此时便可以一次性全部计算小和，不用一个个遍历。

编程复现：

def smallSum(arr):
    # 当数组为空或者只有一个数，小和为0
    if not arr or len(arr) < 1:
        return 0
    return mergeSum(arr, 0, len(arr)-1)

总结：

使用归并排序算法的快速之处在于，归并的两个部分对内都是有序的，因此在比较大小数，可以迅速确定整批的数据大小，而不用重复遍历计算。

(2)逆排序问题

在数组中的两个数字，如果前面一个数字大于后面的数字，则这两个数字组成一个逆序对。如数组[7，6，5，4]中，一共有5个逆序对，分别是（7，6）（7，5）（7，4）（6，4）（5，4）

方法1：遍历每个元素，逐个比较该数字和它后面的数字的大小。如果后面的数字比它小，则这两个数字就组成一个逆序对。该算法时间复杂度为O(n2)。

方法2：

基于归并排序，递归，在排序过程中统计逆序对个数。

逆序对的总数 = 左边数组中的逆序对的数量 + 右边数组中逆序对的数量 + 左右结合成新的顺序数组时中出现的逆序对的数量

先把它分解成两个长度为2的子数组（如a），再把这两个子数组分别拆分成两个长度为1的子数组（如b）。

接下来边合并相邻的子数组，一边统计逆序对的数目。在第一对长度为1的子数组{7}、{5}中，7大于5，因此（7,5）组成一个逆序对，N=1。

同样，在第二对长度为1的子数组{6}、{4}中，也有逆序对（6,4），N+1=2。

由于我们已经统计了这两对子数组内部的逆序对，因此需要把这两对子数组排序（如c），以免在以后的统计过程中再重复统计。

如何对两对子数组排序并再排序过程中更新N?

  1. 设置两个指针分别指向两个子数组中的最大值，p1指向7，p2指向6
  2. 比较p1和p2指向的值，如果p1大于p2，因为p2指向的是最大值，所以第二个子数组中有几个元素就有几对逆序对(第二个子数组当前有两个元素，逆序对加2,N=N+2=4)，7>6,比较完之后将p1指向的值放入辅助数组里，辅助数组里现在有一个数字7，然后将p1向前移动一位指向5
  3. 再次判断p1和p2指向的值，p1小于p2，因为p1指向的是第一个子数组中最大值，所以子数组中没有能和当前p2指向的6构成逆序对的数，将p2指向的值放入辅助数组，并向前移动一位指向4，此时辅助数组内为6,7
  4. 继续判断p1(指向5)和p2(指向4)，5>4,第二个子数组中只有一个数字，逆序对加1，4+1=5。然后将5放入辅助数组，第一个子数组遍历完毕，只剩下第二个子数组，当前只有一个4，将4也放入辅助数组，函数结束。辅助数组此时为4,5,6,7.逆序对为5.

辅助数组作用：用于保存排序好的结果，在递归排序的时候要用到上一步归并的结果。

递归公式的时间复杂度通式：

T(N) = aT(N/b) + O(nd)

(1)如果log(b,a) > d，则时间复杂度为O(Nlog(b,a))

(2)如果log(b,a) = d，则时间复杂度为O(Nd * logN)

(3)如果log(b,a) < d，则时间复杂度为O(Nd)
