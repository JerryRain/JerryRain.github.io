ubuntu进不去recovery mode

Ubuntu进不去recovery mode


问题描述：

开机进入Ubuntu系统，可是开机一直卡在Lenovo这个登录界面进不去，所以决定从Ubuntu的高级选项进入Ubuntu系统的恢复模式(recovery mode)，可是进入恢复模式时，报错，显示VFS:Unable to mount root fs on unknown-block(0,0)，感觉系统都没救了…

如下图所示：

问题分析：

系统内核出现问题，应该是当前内核出现问题导致出问题，删除当前内核可以解决问题

问题解决记录：

通过Ubuntu高级选项，选择内核版本比较低的模式登录系统，进入系统后，打开命令行工具，查看当前内核：

uname -a

查看机器上安装的所有内核：

dpkg --list purge linux-image

输入指令后会显示电脑当前系统所使用的所有内核版本，一般电脑使用的都是最新的内核版本，因为我们可以从低版本内核进入系统，所以出问题的一般就是现在使用最新的内核。

当知道要删除哪些内核时，使用下面的指令删除内核：

sudo apt-get purge linux-image-XXX-generic
sudo apt-get purge linux-image-extra-XXX-generic #删除内核后查看内核会发现有个叫extra的内核，也要把这个内核删除

删除内核之后，运行下面的命令更新grub2：

sudo update-grub2

最后，重启机器：

sudo reboot

开机重启之后就可以直接通过第一个Ubuntu选项登录Ubuntu系统啦～～～
